# Design References

These mockups are approved visual references for implementation direction. Use them as design targets, not as exact production screenshots.

## Approved References

- `homepage-clean-marketplace-reference.png`: primary homepage and marketing-page direction. Use this for clean, bright, polished marketplace composition with generous white space.
- `catalog-browsing-reference.png`: primary pet catalog/index direction. Use this for the busier browsing experience with a strong filter sidebar, top banner, sorting controls, and dense pet grid.
- `process-clean-spacious-reference.png`: approved How It Works page direction. Use this for the calmer marketing-page spacing, clear hero, simple timeline, and restrained checklist sections.
- `pricing-clean-spacious-reference.png`: approved pricing page direction. Use this for the centered clean hero, airy pricing cards, FAQ preview, and generous marketing-page spacing.
- `warranty-header-cards-reference.png`: approved Guarantee page reference for the hero/header area and the three summary boxes: Covered-ish, Not covered, and Please do not send.
- `warranty-form-lower-reference.png`: approved Guarantee page reference for the claim form, lower content flow, FAQ area, and final CTA.

## Usage Notes

- Treat these PNGs as directional design targets, not exact production screenshots.
- Match spacing, hierarchy, mood, and component relationships before matching incidental generated details.
- Marketing pages should follow the cleaner homepage reference rather than the busier catalog reference.
- The How It Works page should follow the process reference while preserving the broader homepage-style white space and calm section rhythm.
- The pricing page should follow the pricing reference and stay calmer than the pet catalog.
- The Guarantee page should combine the header and three summary boxes from `warranty-header-cards-reference.png` with the form and lower-page structure from `warranty-form-lower-reference.png`.
- The pet catalog/index page can be denser because filtering, sorting, and comparison benefit from more interface controls.
- Adapt layouts responsively instead of forcing desktop mockup proportions onto mobile.
- Do not treat out-of-scope elements in generated mockups, such as favorites, saved items, messaging, accounts, or notifications, as approved features.
- Replace generated placeholder copy, icons, and incidental UI with scoped project content and approved features.
- Preserve the Pet Share visual system: rounded typography, bright matte accents, soft glass surfaces, no hard borders, strong pet imagery, and restrained satirical copy.
