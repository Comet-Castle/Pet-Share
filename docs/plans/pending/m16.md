## Environment variables

- [ ] **Audit `.env` / `.env.example` and clean up anything no longer needed.** Cross-check every variable in `.env.example` against what the code actually reads (env parsing/validation in `config/` or `lib/env/`, plus direct `process.env.*` usage) and against the live integrations — Sanity (project ID, dataset, API version, read/browser tokens, preview/webhook secrets), Mailgun, and any Vercel-specific vars. Remove stale/unused entries, confirm each remaining var is documented as required/optional and public vs server-only, and confirm no secret is exposed via a `NEXT_PUBLIC_` prefix. Keep `.env.example` in sync with the final set. Ties into the existing M16 scope items "Confirm Vercel environment variables" and "Required deployment environment variables are documented but not exposed."
