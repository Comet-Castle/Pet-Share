## Entry Animation Expansion (scoping, pre-implementation)

This section scopes closing that gap with subtle fade/slide-up-on-entry motion, site-wide. Not yet implemented ‚Äî outline only.

Relevant precedent from `docs/dependency-decision-log.md:38-39`: Tailwind/CSS-only animation is the default, and Framer Motion is the "preferred escalation path... if scroll-triggered patterns become common," with an explicit warning to **not build a broad custom Intersection Observer animation framework before considering Framer Motion instead**. ### Focus states (keyboard vs mouse)

Reported: clicking interactive elements with a **mouse** (logo, nav/menu links, accordion triggers, pet cards) leaves a lingering focus outline/ring ‚Äî orange on some systems. Two causes: (1) elements that set `focus:` (fires on both mouse and keyboard) ‚Äî e.g. `components/layout/site-nav-links.tsx:34` (`focus:ring-pet-coral`), `components/ui/button.tsx:28` (`focus:ring-pet-blue`), `cms-form.tsx`; (2) elements with no override falling back to the **browser default outline** (accordions, `pet-card.tsx`), which renders orange.

Do **not** remove focus visibility outright ‚Äî WCAG 2.4.7 "Focus Visible" and the CLAUDE.md Accessibility Standards require a visible keyboard focus state. This is not a user-preference toggle; the compliant fix auto-detects input mode via `:focus-visible` (keyboard shows the ring, mouse does not).

- [x] Migrated `focus:outline-none focus:ring-*` ‚Üí `focus-visible:outline-none focus-visible:ring-*` across all interactive components under `components/` and `app/` (button, nav, form, drawers, filters, gallery, cards, carousel, portable-text, etc.). Keyboard users keep the branded ring; the ring no longer lingers after mouse clicks.
- [x] Added a global `:focus-visible` reset + branded coral ring in `app/globals.css` (with `:focus:not(:focus-visible) { outline: none }`) so any focusable fallback without an explicit Tailwind override gets a consistent keyboard-only ring instead of the browser-default outline.
- [x] Needs-visual-check: Tabbed through logo, nav/menu, accordions, pet cards, form fields, and drawers ‚Äî the ring is clearly visible on keyboard focus; clicking each with a mouse leaves no lingering ring (`:focus-visible` behaving correctly). Ring contrast against its background meets the non-text contrast threshold.
