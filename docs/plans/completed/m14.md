- [x] **Ran a full preview** and verified media coverage before writing.
- [x] **Ran the confirmed write with purge** (`pnpm seed:sanity -- --confirm --purge`). Hit and fixed two real bugs along the way (see below), then completed successfully: 396 media assets uploaded, 148 documents seeded (53 pet types, 12 owners, 50 pets, 20 testimonials, 3 forms, 5 pages, singletons/system pages).
  - **Bug 1 — purge chunking crossed type boundaries.** `deleteMutationChunkSize`-based chunking could put a referencing document (e.g. `homePage`) and a document it references (e.g. a `pet`) in the *same* Sanity transaction; Sanity validates references against already-committed state, not other deletes in the same transaction, so the delete was correctly rejected. Fixed by chunking strictly within each type, in dependency order, awaiting each type's deletion before starting the next (`purgeSeedDocuments()` in `scripts/seed-sanity.mjs`).
  - **Bug 2 (real, not code) — stale Studio drafts blocked the purge.** Multiple genuine unpublished drafts existed in Sanity Studio (a `homePage` draft, a pet draft, and 3 marketing-page drafts, dated late June–early July), invisible to normal GROQ queries due to Sanity's default "published" query perspective silently filtering out drafts — confirmed only via `perspective: "raw"` and `client.getDocument()`. These drafts referenced documents we were purging. Per explicit user decision, all 5 blocking drafts were deleted (unrelated `sanity.previewUrlSecret` system drafts were left alone).
  - **Minor side-effect of the full media wipe:** the 3 hand-crafted `customIconAssetKey` SVGs (dog/cat/rabbit) aren't part of the AI-generation pipeline (`buildMediaPrompts()` doesn't cover them) and were wiped along with everything else. They were non-destructively recovered from git history (commit `ee0d8fc`, before the M13 untracking commit) and re-uploaded; all 3 pet types now have their custom icon linked again. Confirmed via query.
- [x] **Confirmed relationships, slugs, references, image roles, and page-builder sections resolve correctly.** Verified via direct Sanity queries: 0 dangling `petType`/`owner` refs on pets, 0 pets/owners missing images, `homePage.featuredPets`/`testimonials`/`heroCarousel` all fully resolved, exactly 4 marketing page slugs present (`how-it-works`, `pricing`, `contact`, `guarantee`) with no `about`.
